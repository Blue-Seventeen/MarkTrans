现在整个项目的核心逻辑部分已经完成，我现在需要一套可以操作的 UI 来完善整个界面，
# 0x01: 功能清单
1. 用户可以选择导入的 markdown 文件的本地位置
2. 用户可以选择本地存储 markdown 图片的目录（Obsidian 中的附件资源会被专门设置存储到一个文件夹中）
3. 用户可以直接输入 Markdown 内容
4. 界面获取到用户输入的 Markdown 内容后，会调用 markdown_ast_parser 中的相关方法，将其转换为 AST 树，并显示在前端界面上
5. 界面在获得 AST 数据后，会调用 ast_html_translator 中的相关方法将其转换为 HTML 格式，并显示到前端界面上。

6. 用户可以通过选择来切换 AST 转 Markdown 的 Style（这部分功能通过 mapping_style 表控制，is_active 代表当前启用的样式，is_deletable 代表该样式是否允许删除，对于默认风格，用户只能编辑 style_name 不可以对其删除，其余用户后添加的 style 均可删除）
7. 用户可以选择新增 ”风格“，在新增风格前，用户可以选择基于 ”XXX“ 新增（要求用户输入新增风格的名称），比如基于 ”默认风格“ 新增，此时系统会先 Copy mapping_rule 表中与默认风格有关的所有数据，并追加到 mapping_rule 表，同时设置 mapping_style 表新增的风格 is_active = 1

8. 界面提供便捷的数据库管理视图，用户可以选择编辑的数据库列表
9. 针对每个数据库列，用户可以自定义选择依据哪列进行升序或者降序排列
10. 针对每个数据库表，界面提供添加功能，且在添加时，id 字段自动填上去（默认为数据库中已有 id 最大值 + 1，如果后续因为修改导致 id 断层，这个无所谓）
11. 针对数据库表中的每个项，允许用户进行编辑与删除操作，编辑时 id 字段不允许用户编辑

# 0x02：用户角色与使用场景
用户主要是可以通过我们的界面，知道 Markdown => HTML 的实际转化流程，并且可以通过操作数据库很快速方便的修改转换逻辑。
我期待，Markdown 输入，AST 输出，HTML 输出，以及对 Style 的 选择和编辑在同一页上完成，数据库管理可以新增个导航栏在另一页操作。
这个项目主要是在电脑上使用，只要适配电脑端即可，手机平板无所谓，不用管。
